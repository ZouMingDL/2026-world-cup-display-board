# Plan: 获取2026年世界杯真实数据

## 背景
用户希望将当前硬编码的静态数据替换为真实的2026年世界杯数据，包括：
1. 参赛国家队的FIFA世界排名
2. 球员真实基础信息（俱乐部、身价、年龄等）
3. 球员排名数据

## 数据源调研结果

### 可用的免费API

**1. TheSportsDB API (推荐)**
- 免费API，无需注册即可使用基础功能
- 已确认有2026年世界杯比赛数据（联赛ID: 4429）
- 提供：球队信息、队徽/Logo、比赛赛程
- API端点示例：
  - `https://www.thesportsdb.com/api/v1/json/3/eventsseason.php?id=4429&s=2026` - 获取比赛列表
  - `https://www.thesportsdb.com/api/v1/json/3/searchteams.php?t=argentina` - 搜索球队

**2. Football-Data.org API**
- 需要API Key（免费套餐有限制）
- 提供FIFA排名、比赛数据
- 当前返回403/404，可能需要注册获取Key

**3. FIFA官方**
- 网页动态加载，无法直接抓取
- 需要浏览器自动化工具

### 数据缺口
- **球员详细数据**：免费API不提供球员详细信息（身价、能力值等）
- **FIFA排名**：需要手动维护或使用付费API
- **球员雷达图数据**：需要根据真实数据估算或使用FIFA游戏数据

## 实施方案

### 方案：混合数据获取（推荐）

#### 第一步：创建数据获取脚本
创建 `src/scripts/fetchWorldCupData.ts`，使用 TheSportsDB API 获取：
- 48支参赛国家队信息（名称、队徽、所属大洲）
- 小组分组信息
- 比赛赛程

#### 第二步：手动补充FIFA排名数据
由于免费API不提供FIFA排名，创建 `src/data/fifaRankings.ts`：
- 基于2026年4月最新FIFA排名（可从公开资料获取）
- 手动维护排名数据

#### 第三步：球员数据来源
方案A（推荐）：使用公开的球员数据库
- 从Transfermarkt等网站手动收集关键球员数据
- 创建 `src/data/playerDatabase.ts` 存储真实球员信息
- 包括：真实姓名、俱乐部、年龄、身价、位置

方案B：使用FIFA游戏数据
- 如果有FIFA游戏数据文件，可以解析获取球员能力值

#### 第四步：更新worldCupData.ts
- 修改数据结构以支持真实数据
- 保持与现有UI组件的兼容性
- 保留现有图片占位符逻辑（用于无头像时）

2. 检查是否正确显示48支参赛队
3. 验证FIFA排名显示正确

### 限制说明

- 球员能力值（雷达图数据）可能需要估算，因为免费API不提供
- 比赛结果需要在比赛开始后实时更新
- 部分小国球队的球员数据可能不完整
